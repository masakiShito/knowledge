# Knowledge Repository

このリポジトリは、**個人用ナレッジベース（Knowledge Base）**として運用されています。  
日々の調査結果、設計メモ、技術検証、脆弱性情報、考察などを **体系的に蓄積・再利用** することを目的としています。

---

## 📚 目的（Purpose）

- 技術・設計・調査結果を **忘れないために残す**
- 「調べ直し」を減らし、**思考の再利用性を高める**
- 将来の自分・チームへの **知識の引き継ぎ資産** とする

本リポジトリは「完成されたドキュメント」ではなく、  
**成長し続ける知識のログ（Living Knowledge）** です。

---

## 🧠 管理方針（Concept）

- ナレッジは **Markdown（.md）** で記述する
- Obsidian を用いてローカルで編集・リンク管理を行う
- 完璧さよりも **記録することを優先**
- 後からの整理・再構成を前提とする

---

## 🗂 コンテンツ例（Contents）

- 技術メモ（Frontend / Backend / Infrastructure）
- 設計・アーキテクチャ考察
- セキュリティ・脆弱性情報
- ライブラリ・フレームワークの調査結果
- 業務で得た知見・失敗談・ベストプラクティス

※ 内容は時系列・粒度ともに不均一であることがあります。

---

## 🔄 運用ルール（Operation）

- 基本は **Obsidian から自動コミット**
- コミットメッセージは自動生成される場合があります
- コミット単位は「意味の塊」ではなく **変更検知ベース**

例：
```text
docs: update knowledge notes
chore: sync obsidian notes
