以下、**2025年12月に公表・悪用が広がった React（React Server Components / RSC）周りの重大脆弱性**を、ナレッジとしてそのまま貼れる形でまとめます（深掘り版）。中心は通称 **React2Shell（CVE-2025-55182）** です。 

---

## **1. 何が起きたか（結論）**

- **RSC（React Server Components）の “Flight protocol” 実装に起因する、認証不要のRCE（リモートコード実行）**が見つかった（CVE-2025-55182 / CVSS 10.0）。 
    
- **影響は React 19 系の RSC 関連パッケージ**（react-server-dom-*）に及び、**Next.js などRSCを採用するフレームワーク**が巻き込まれる形で注意喚起が出た。 
    
- 公表直後から**実際の悪用（in-the-wild）が観測**され、脅威インテリジェンス各社が追跡記事を出している。 
    

---

## **2. 影響範囲（どれが危ない？）**

### **影響を受けるパッケージ（React公式）**

脆弱性（CVE-2025-55182）は、少なくとも以下に存在： 
- react-server-dom-webpack
- react-server-dom-parcel
- react-server-dom-turbopack
  
### **影響バージョン（React公式）**

- 影響あり：**19.0 / 19.1.0 / 19.1.1 / 19.2.0** 
- 修正版：**19.0.1 / 19.1.2 / 19.2.1**（少なくともこのラインへ更新が推奨） 
### **追加で出てきた関連脆弱性**

12月中旬にかけて、**DoS** や **ソースコード露出**に絡む追加のCVEも注意喚起（CVE-2025-55183 / CVE-2025-55184 / CVE-2025-67779 など）。 

---

## **3. 技術的な肝（なぜRCEになる？）**

- 根本はざっくり言うと **「サーバ側で扱うRSC通信（Flight）の処理で、攻撃者が送ったデータを危険な形でデシリアライズしてしまう」** こと。結果、**任意コード実行に繋がりうる**。 
- 重要ポイント：**自分のアプリが明示的に “React Server Function endpoint” を実装していなくても、RSCをサポートしているだけで影響し得る**、という注意が公式から出ている。 

（※詳細なペイロードや再現手順の提供は、悪用を助けるのでここでは書かない方針でOK。社内ナレッジとしても“防御側の要点”に寄せるのが安全です。）
